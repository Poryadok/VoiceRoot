# Federation — федерация серверов

## Концепция

Voice поддерживает федерацию: любой человек может поднять свой Voice-сервер (federated node),
который становится частью глобальной сети Voice.

Это даёт:
- **Приватность**: сообщения и файлы спейса хранятся на ноде, не у главного сервера
- **Снижение расходов**: войс, стримы и хранилище спейсов — за счёт владельца ноды
- **Конкурентное преимущество**: Discord не позволяет self-hosting

## Роли серверов

| | Master (главный сервер) | Federated node |
|---|---|---|
| Аккаунты пользователей | Все аккаунты здесь | Нет |
| Авторизация | Всегда через master | Нет |
| DM | Живут здесь | Нет |
| Спейсы | Может хостить | Хостит свои спейсы |
| Файлы спейса | Cloudflare R2 | Своё хранилище |
| Войс / стримы | Свой SFU | Свой SFU |
| Глобальный каталог | Индексирует всё | Нет |
| Матчмейкинг | Здесь | Нет |

Аккаунты существуют только на master. Federated node хостит спейсы, но не создаёт аккаунты.

## Поток авторизации

1. Пользователь заходит в спейс на federated ноде
2. Нода отправляет запрос к master: "Это @username, у него есть доступ?"
3. Master отвечает ✓/✗ + роль пользователя в спейсе
4. Нода пропускает или блокирует

## Регистрация ноды

- По заявке (не открытая регистрация) — чтобы контролировать качество на старте
- Владелец принимает ToS при регистрации
- Master выдаёт сертификат доверия (signed token)
- Нода включается в глобальный каталог спейсов

## Модерация

Модель как App Store: хочешь быть в сети — играй по правилам.

- Нода несёт ответственность за контент на своих спейсах (прописано в ToS)
- Пользователи репортят контент через стандартную кнопку → жалоба идёт и владельцу ноды, и в основную команду
- Нарушение ToS → **дефедерация**: нода отключается от сети, её спейсы пропадают из каталога
- Нелегальный контент (CSAM и т.п.) — немедленная дефедерация без предупреждения

## Технические требования

Главное архитектурное решение: **спроектировать Server-to-Server (S2S) API в V1**.

Основной сервер реализует обе роли:
- Client-facing API (приложения пользователей)
- S2S API (взаимодействие с federated нодами)

Federated node = тот же Voice backend, зарегистрированный в мастере.

## V1 scope

**Включено:**
- Federated спейсы (текстовые и войс-чаты на ноде)
- Авторизация через master
- Глобальный каталог с federated спейсами
- Репорты и дефедерация

**Не включено в V1:**
- DM между пользователями разных серверов (все DM — на master, проблемы нет)
- Открытая регистрация нод (только по заявке)
- Аккаунты на federated нодах
