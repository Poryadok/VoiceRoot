# Text Chat — текстовый чат

## Сообщения

- **Редактирование**: да, лимит 100 лет (фактически без ограничений)
- **Удаление**: да, у всех, без лимита по времени
- **Форматирование**: Discord-стиль
  - bold `**text**`, italic `*text*`, underline `__text__`, strikethrough `~~text~~`
  - spoiler `||text||`
  - inline code `` `code` ``
  - code block ` ```language ` с подсветкой синтаксиса
  - заголовки `# ## ###`
  - списки `- / 1.`
  - цитаты `>`

## Треды и ответы

| Тип чата | Режим |
|----------|-------|
| DM (личка) | только reply на конкретное сообщение |
| Канал | только треды — у каждого сообщения есть тред, в котором можно отвечать |
| Группа | треды включаются/выключаются в настройках группы |

## Медиа и вложения

- Файлы и изображения: да — лимиты и хранилище см. [file-storage.md](file-storage.md)
- Превью ссылок: да, автоматически — запрос делает **клиент**
- Голосовые сообщения: да (аудиофайл + встроенный плеер)
- GIF: да
- Стикеры: да — системные паки + пользовательские (загрузка своих паков)
- Emoji: да

## Социальные механики

- Реакции на сообщения: да
- @упоминания: да (завязаны на систему нотификаций — TBD отдельно)
- Закреплённые сообщения: да, как в Telegram

## История и поиск

- История: бесконечная (лимит не установлен на старте, пересмотреть по нагрузке)
- **Поиск из чата** → поиск по тексту и ссылкам внутри этого чата
- **Поиск с главного экрана** → глобальный поиск по всем чатам
- У каждого чата есть раздел shared media: отдельные вкладки для фото / видео / файлов / ссылок

## Статусы доставки

- Как в Telegram: одна галочка = доставлено, две = прочитано
- В группах/каналах — счётчик просмотров (как в Telegram)

## Хранение

- Все сообщения и файлы хранятся на сервере
- Детали файлового хранилища — [file-storage.md](file-storage.md)

## Лимиты

- Максимальная длина сообщения: **4000 символов** (Telegram: 4096, Discord: 2000 / 4000 Nitro)

## Технические решения

- **Протокол**: WebSocket, persistent connection; reconnection: exponential backoff (1s → 2s → 4s, cap 30s)
- **Catch-up**: при reconnect клиент шлёт `last_message_id`, сервер отдаёт пропущенное из DB; offline queue не нужна
- **Между инстансами**: Redis Pub/Sub
- **Typing indicator**: WebSocket, throttle — событие не чаще раза в 3 сек, гасить через 5 сек без обновления
- **Read receipts**: DM — галочки (одна = доставлено, две = прочитано); группы/каналы — только счётчик непрочитанных для самого пользователя, без раскрытия кто прочитал (как Discord)
- **Rate limiting**: 5 сообщений / 5 сек на пользователя (глобально); slow mode для каналов — 5 сек – 6 ч, настраивается админом спейса
- **Реакции**: стандартные Unicode + кастомные эмодзи спейса (как Discord)
- **Лимит группы**: 500 участников для обычной группы (не спейса); для больших сообществ — спейс

## Пост-v1 backlog

- **Запланированные сообщения** — отправка в заданное время (как в Telegram); отображаются отдельной плашкой над строкой ввода, редактируются и отменяются до момента отправки
