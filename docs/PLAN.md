# Voice — План разработки

> Каждая фаза — минимальный инкремент, который даёт пользователю ощутимый прогресс.
> Фичи внутри фазы упорядочены по приоритету реализации.

---

## Текущее состояние кодовой базы

| Компонент | Состояние |
|-----------|-----------|
| VoiceAuthService (Java/Spring) | Работает: регистрация, логин, JWT, роли |
| VoiceWebSocketService (Go) | Заглушка: gRPC-сервер стартует, методы не реализованы. В целевой архитектуре real-time шлюз — **Realtime Service** ([MICROSERVICES.md](MICROSERVICES.md)) |
| VoiceChannelDataService (Go) | Пустой: только go.mod |
| VoiceMessageService (Go) | Пустой: только go.mod |
| voiceclient (Flutter) | Прототип: P2P WebRTC через Firebase, не подключён к бэкенду |
| Инфраструктура | Docker для Auth и WS; нет CI/CD, нет схем БД |

## Фаза 0 — Фундамент

**Цель:** подготовить инфраструктуру, без которой невозможна разработка фич.

- [ ] **Схемы базы данных** — спроектировать и создать миграции для всех сервисов MVP (users, messages, conversations, friends)
- [ ] **API Gateway** — единая точка входа для клиента (REST + WebSocket); маршрутизация на внутренние сервисы
- [ ] **CI/CD pipeline** — GitHub Actions: lint, тесты, сборка Docker-образов, деплой на staging
- [ ] **Docker Compose для dev** — одна команда поднимает все сервисы + PostgreSQL + Redis локально
- [ ] **VoiceAuthService: доработка** — refresh tokens (opaque, 30 дней), logout, валидация входных данных, тесты
- [ ] **Общая библиотека Go** — JWT-валидация, middleware авторизации, логирование, конфиг — для всех Go-сервисов
- [ ] **Flutter-клиент: скелет** — навигация (three-column layout), подключение к бэкенду вместо Firebase, DI, state management

**Результат:** все сервисы стартуют локально, клиент авторизуется через бэкенд.

Инженерные договорённости (ветки, PR, тесты, окружения, выкат): [CONTRIBUTING.md](CONTRIBUTING.md), [TESTING.md](TESTING.md), [DEPLOYMENT.md](DEPLOYMENT.md).

## Фаза 1 — MVP: Личные сообщения

**Цель:** пользователь может зарегистрироваться, найти друга и переписываться 1-на-1.

### Бэкенд

- [ ] **VoiceMessageService** — CRUD сообщений: отправка, получение, редактирование, удаление
  - Хранение в PostgreSQL, пагинация курсором
  - REST API для CRUD
- [ ] **VoiceWebSocketService** (в продукте — **Realtime Service**) — real-time доставка
  - WebSocket подключение с JWT-аутентификацией
  - Доставка новых сообщений подключённым клиентам
  - Redis Pub/Sub для маршрутизации между инстансами
  - Reconnect: нумерация событий `s`, op `resume`; догрузка пропущенных сообщений курсором через **Messaging** (не через WS-шлюз) — [ARCHITECTURE_REQUIREMENTS.md](ARCHITECTURE_REQUIREMENTS.md)
- [ ] **VoiceChannelDataService** — управление чатами
  - Создание DM-диалогов
  - Список диалогов пользователя с последним сообщением и счётчиком непрочитанных
  - `mark_read` — отметка прочитанного
- [ ] **Друзья и контакты (базовый)** — запрос дружбы, принятие/отклонение, список друзей, блокировка
- [ ] **Профиль пользователя (базовый)** — отображаемое имя, аватар (загрузка в R2), статус "О себе"
- [ ] **Присутствие (базовый)** — онлайн / оффлайн (по WebSocket-подключению), последний визит

### Клиент (Flutter Web)

- [ ] Экраны регистрации и логина
- [ ] Список диалогов с превью последнего сообщения
- [ ] Экран чата: отправка/получение сообщений в реальном времени
- [ ] Поиск пользователей, отправка запроса дружбы
- [ ] Профиль: просмотр и редактирование
- [ ] Индикатор онлайн/оффлайн

### Не входит в Фазу 1

- Форматирование Markdown, реакции, треды, пины
- Вложения (файлы, картинки)
- Typing indicator
- Push-уведомления
- Голосовые звонки

**Результат:** работающий веб-мессенджер для 1-на-1 переписки. Можно звать первых тестеров.

## Фаза 2 — Голосовые звонки и медиа

**Цель:** пользователь может позвонить другу и отправить картинку в чат.

### Голос

- [ ] **LiveKit интеграция** — развернуть LiveKit Server, сгенерировать токены на бэкенде
- [ ] **1:1 звонки** — инициация звонка, входящий звонок (WebSocket-событие), принятие/отклонение
- [ ] **Клиент: UI звонка** — экран звонка, mute, speaker, завершение
- [ ] **Signaling через бэкенд** — вместо Firebase, передача offer/answer/ICE через WebSocket

### Медиа и вложения

- [ ] **Загрузка файлов** — Cloudflare R2, presigned URLs, лимит 50MB (free)
- [ ] **Вложения в сообщениях** — картинки (thumbnail + полный размер), файлы (иконка + имя + размер)
- [ ] **Сжатие изображений** — автоконвертация в WebP, генерация thumbnail

### Улучшения чата

- [ ] **Typing indicator** — "печатает...", throttle 3 сек
- [ ] **Статусы доставки** — отправлено / доставлено / прочитано (галочки)
- [ ] **Редактирование и удаление сообщений** — с отметкой "(ред.)", удаление для всех/для себя

**Результат:** полноценный 1:1 мессенджер с голосом и медиа. Конкурент Telegram по базовому функционалу.

## Фаза 3 — Групповые чаты

**Цель:** общение компанией — текст и голос.

- [ ] **Групповые текстовые чаты** — создание группы (до 200 участников), добавление/удаление участников, аватар группы
- [ ] **Роли в группе (простые)** — создатель (owner) и участники; owner может удалять людей и менять название
- [ ] **Групповые голосовые звонки** — до 32 участников, любой может присоединиться к активному звонку группы
- [ ] **Пересылка сообщений** — forward из одного чата в другой с атрибуцией
- [ ] **Реакции на сообщения** — набор эмодзи, счётчик, список реагировавших
- [ ] **Уведомления (базовые)** — in-app уведомления, звук, badge-счётчик

**Результат:** можно собрать компанию друзей в одном чате. Продукт становится полезнее Telegram для гейминг-компаний.

## Фаза 4 — Спейсы (базовые)

**Цель:** аналог серверов Discord — сообщества с каналами.

### Спейсы

- [ ] **Создание спейса** — название, иконка, описание
- [ ] **Каналы** — текстовые и голосовые каналы внутри спейса
- [ ] **Категории** — группировка каналов
- [ ] **Инвайт-ссылки** — генерация, срок действия, лимит использований
- [ ] **Ролевая модель** — Owner → Admin → Moderator → Member → Guest; назначение ролей
- [ ] **Базовая модерация** — кик, бан, мьют, slow mode

### Клиент

- [ ] **Sidebar со спейсами** — список спейсов пользователя, переключение
- [ ] **Список каналов** — дерево категорий и каналов внутри спейса
- [ ] **Войс-каналы** — постоянные голосовые комнаты (join/leave, список участников)

### Улучшения

- [ ] **Текстовый чат: Markdown** — жирный, курсив, код, цитаты, ссылки
- [ ] **Упоминания** — @username, @everyone, @here
- [ ] **Пины** — закрепление важных сообщений

**Результат:** полноценная альтернатива Discord для небольших сообществ. Можно продвигать как "Discord без блокировок".

## Фаза 5 — Матчмейкинг

**Цель:** ключевое отличие от конкурентов — поиск тиммейтов.

- [ ] **Каталог игр** — предзаполненный список (Dota 2, CS2, Valorant, PUBG и т.д.) с конфигурацией ролей/рангов
- [ ] **Профиль игрока** — выбранные игры, self-reported ранг и роли, регион
- [ ] **Очередь матчмейкинга** — поиск по критериям (игра, ранг, роль, регион, количество слотов)
- [ ] **Подбор и уведомление** — при наборе полной группы создаётся временная комната (текст + голос), push всем участникам
- [ ] **Таймаут** — 30 минут без подтверждения → выброс из очереди
- [ ] **Оценка напарников** — звёзды 1-5 после матча, средний рейтинг в профиле
- [ ] **История матчей** — список прошлых подборов в профиле

**Результат:** уникальная фича, которой нет у конкурентов. Главный аргумент для привлечения геймеров.

## Фаза 6 — Push-уведомления и мобильные платформы

**Цель:** выход за пределы веба, мобильные пользователи.

### Push

- [ ] **FCM** — Android + Web
- [ ] **APNs** — iOS (обычные уведомления)
- [ ] **APNs VoIP** — PushKit + CallKit для входящих звонков на iOS
- [ ] **Группировка** — один push на чат, обновление существующего

### Мобильный клиент

- [ ] **Адаптация Flutter UI** — мобильный layout (single-column с навигацией)
- [ ] **Сборка Android** — APK / Play Store
- [ ] **Сборка iOS** — TestFlight → App Store
- [ ] **Оффлайн-кэш** — последние 50 сообщений на канал (SQLite)

### Desktop

- [ ] **Windows-сборка** — Flutter desktop, auto-updater

**Результат:** мультиплатформенный продукт. Основная аудитория (мобильные геймеры) получает доступ.

## Фаза 7 — Поиск, треды, шара экрана

**Цель:** улучшения для активных сообществ.

- [ ] **Полнотекстовый поиск** — PostgreSQL tsvector/GIN (v1), поиск по сообщениям внутри чата и глобально
- [ ] **Поиск пользователей и спейсов** — глобальный каталог публичных спейсов
- [ ] **Треды** — ответы в ветке, обязательные в каналах, опциональные в группах
- [ ] **Шара экрана** — демонстрация экрана/окна в войс-сессиях, до 3 одновременных стримов
- [ ] **Shared media tabs** — вкладки "Медиа / Файлы / Ссылки" в инфо чата
- [ ] **Кастомные роли** — создание ролей с гранулярными правами (32+ типа), channel-level overrides

**Результат:** продукт готов для крупных сообществ с сотнями участников.

## Фаза 8 — Монетизация

**Цель:** начать зарабатывать.

### Подписка

- [ ] **Premium ($5/мес)** — кастомный статус, анимированный аватар, баннер профиля, увеличенные лимиты файлов (200MB), 5 профилей
- [ ] **Space Pro ($5/мес)** — до 5000 участников, 128 слотов голосового канала, кастомные эмодзи
- [ ] **Приём платежей** — Paddle (международные), CloudPayments (РФ)
- [ ] **Годовая подписка** — скидка 20%

### Множественные профили

- [ ] **Переключение профилей** — 2 бесплатных / 5 платных, независимые контакты и спейсы

### Верификация

- [ ] **Личная** — привязка Twitch/YouTube, значок в профиле
- [ ] **Организации** — верификация через DNS TXT-запись

**Результат:** появляется revenue stream. Freemium без ограничений базового функционала.

## Фаза 9 — Безопасность и модерация

**Цель:** готовность к масштабированию аудитории.

- [ ] **Репорты** — жалобы на пользователей/сообщения/спейсы, категории (спам, оскорбления, читы)
- [ ] **Авто-модерация** — shadow-ban при ≥10 жалоб/24ч или 1% порог
- [ ] **Модераторская панель** — веб-интерфейс для глобальных модераторов
- [ ] **2FA** — TOTP (Google Authenticator и т.д.)
- [ ] **E2E шифрование (DM)** — опциональное, Signal Protocol, с ограничениями поиска
- [ ] **ClamAV** — антивирусная проверка загружаемых файлов
- [ ] **Приватность** — гранулярные настройки видимости полей профиля (три пресета + кастом)

**Результат:** продукт безопасен для массовой аудитории.

## Фаза 10 — Экосистема

**Цель:** привлечение разработчиков и контент-мейкеров.

- [ ] **Боты** — App Manifest (YAML), slash-команды, webhook/polling доставка, rate limits
- [ ] **Deep links** — `voice://` и `https://voice.gg/` для спейсов, каналов, профилей, инвайтов
- [ ] **Сторис** — 24-часовой контент, game tags, "looking for party", архив
- [ ] **Онбординг** — 4-5 шагов для новых пользователей, контекстные подсказки
- [ ] **Accessibility** — клавиатурная навигация, screen reader, контраст, reduced motion

**Результат:** платформа с API для ботов и расширений. Сообщество может строить свои интеграции.

---

## Фаза 11 — Федерация и Self-hosting

**Цель:** корпоративный сегмент и приватность.

- [ ] **S2S Federation API** — gRPC bidirectional stream, авторизация через master
- [ ] **Self-hosted нода** — Docker-образ для развёртывания на своём железе
- [ ] **Синхронизация** — ролей, банов, событий между master и нодами
- [ ] **Дефедерация** — отключение ноды за нарушение ToS
- [ ] **Маршрутизация уведомлений** — нода → master → FCM/APNs

**Результат:** корпоративные клиенты могут развернуть Voice на своей инфраструктуре.
