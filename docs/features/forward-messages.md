# Forward Messages — пересылка сообщений

Поведение как в Telegram.

## Что можно пересылать

- Одно сообщение
- Несколько сообщений (мульти-выбор — как в Telegram: зажать/чекбокс)
- Медиа, файлы, голосовые сообщения, стикеры — с сохранением контента
- Уже пересланные сообщения — атрибуция сохраняется (цепочка "forwarded from" не накапливается — показывается только оригинальный источник)

## Куда можно переслать

- В любой DM (контакт или существующая переписка)
- В любую группу, в которой состоит пользователь
- В любой **текстовый канал** (`chats`, `type = channel`) — в спейсе или **вне** спейса — где есть право на отправку в основной поток (обычно от имени канала / модерации)
- Получатель выбирается через стандартный поиск/список чатов

## Атрибуция ("Forwarded from")

- По умолчанию отображается плашка **"Forwarded from [Имя]"** над пересланным контентом
- Если оригинальный отправитель удалил аккаунт — показывается "Deleted account"
- Если оригинальное сообщение было в закрытом пространстве — атрибуция показывает имя, но без ссылки на источник (нельзя перейти в контекст)
- Пользователь может запретить пересылку своих сообщений в настройках приватности → тогда кнопка Forward недоступна для его сообщений

## Пересылка без атрибуции ("Copy as new message")

- Альтернативное действие рядом с Forward: скопировать содержимое как новое сообщение без плашки "Forwarded from"
- Применимо к тексту; для медиа — пересылает файл без атрибуции

## UX

- Кнопка Forward в контекстном меню сообщения (свайп или долгий тап на мобиле, правая кнопка / hover на десктопе)
- При мульти-выборе: кнопка Forward появляется на панели над клавиатурой/внизу экрана
- После выбора получателя — опциональное поле для добавления своего комментария перед пересланными сообщениями (как в Telegram)

## Что НЕ поддерживается в первой версии

- Пересылка с ограничением "только для друзей" — либо разрешено всем, либо запрещено всем (настройка приватности бинарная)
- Пересылка в несколько чатов одновременно (Telegram позволяет) — отложено
