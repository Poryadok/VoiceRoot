# Bots — бот-платформа

## Концепция

Бот — это приложение стороннего разработчика, которое действует в спейсе как участник: отвечает на slash-команды, отправляет сообщения, реагирует на события. Модель — Discord-style: typed slash commands, interaction-based (бот не читает обычные сообщения без явного права).

---

## App Manifest

Бот описывается декларативным манифестом (YAML или JSON). Манифест содержит всё необходимое для регистрации — можно хранить в репо и деплоить автоматически.

```yaml
name: StatsBot
description: Статистика игроков по CS2, Valorant, Dota 2
icon_url: https://example.com/icon.png
webhook_url: https://example.com/interactions

scopes:
  - send_messages        # отправлять сообщения в каналы
  - send_dm              # писать в DM пользователю (по его инициативе)
  - manage_roles         # выдавать/забирать роли (если нужно)
  - read_members         # видеть список участников спейса

commands:
  - name: stats
    description: Показать статистику игрока
    options:
      - name: game
        type: string
        required: true
        autocomplete: true
      - name: user
        type: user
        required: false
```

**Правило:** платформа не выдаёт боту права, которые не задекларированы в манифесте. Нет implicit admin.

## Slash-команды

### Типы параметров

| Тип | Описание |
|-----|----------|
| `string` | Строка; поддерживает autocomplete |
| `integer` | Целое число |
| `boolean` | Да / Нет (чекбокс) |
| `user` | Выбор участника спейса |
| `channel` | Выбор канала |
| `role` | Выбор роли |
| `attachment` | Файл / изображение |

### Autocomplete

Если параметр помечен `autocomplete: true` — по мере набора клиент отправляет боту запрос, бот возвращает список вариантов (до 25). Например, `/stats game cs` → бот предлагает "CS2", "CS:GO".

### Типы ответов

| Тип | Поведение |
|-----|-----------|
| Обычный | Видят все в канале |
| Ephemeral | Видит только вызвавший команду; не остаётся в истории |
| Deferred | Бот немедленно подтверждает получение ("обрабатываю..."), отвечает асинхронно |

**Обязательный timeout:** бот должен ответить или выслать defer в течение **3 секунд**. Если бот недоступен — клиент показывает грейаут команды и поясняющее сообщение (не безликую "Interaction failed").

### Группировка команд

Поддерживаются подкоманды для группировки: `/queue join`, `/queue leave`, `/queue status`. В `/`-меню они отображаются сгруппированно под именем бота.

## Токены и безопасность

### Bot Token

- Bot token — долгоживущий, не истекает автоматически
- Может быть отозван владельцем вручную через Developer Portal
- Может быть отозван автоматически при нарушениях правил платформы
- При отзыве: генерируется новый токен; все активные webhook-соединения и polling-сессии инвалидируются
- Автоматическая ротация в v1 не предусмотрена (аналогично Discord)

## Добавление бота в спейс

1. Администратор спейса открывает страницу бота (URL вида `voice.app/bots/statsbot`)
2. Видит: описание, запрашиваемые scopes на человеческом языке, список команд
3. Выбирает **каналы**, в которых бот может работать (whitelist; по умолчанию — ни одного)
4. Подтверждает → бот появляется в спейсе

Кто может добавлять ботов — право в [ролевой модели](roles.md) (`manage_bots`). По умолчанию — Администратор и выше.

## Управление ботом в спейсе

### Per-channel контроль

В настройках каждого канала — секция "Боты": список добавленных ботов с тоглами. Администратор может отключить конкретного бота в конкретном канале без удаления из спейса.

В `/`-меню команды бота, недоступного в данном канале, не показываются.

### Статус бота

В списке ботов спейса отображается статус: **онлайн / офлайн**. Если бот офлайн:
- Его команды в `/`-меню грейаутятся
- Тултип: "Бот недоступен"

### Удаление бота

При удалении бота из спейса платформа автоматически удаляет:
- Роли, созданные ботом
- Закреплённые сообщения бота
- Команды бота из `/`-меню

Сообщения, отправленные ботом в каналы, **остаются** (как сообщения удалённого пользователя).

## Доставка interactions боту

### Webhook (v1)

Бот указывает в манифесте `webhook_url`. Платформа POST-ит interaction на этот URL. Бот должен ответить в течение 3 секунд или вернуть defer.

**Подпись запроса:** каждый webhook-запрос подписывается по алгоритму HMAC-SHA256.
- Заголовок: `X-Voice-Signature: v1=<hex>`
- Подпись вычисляется от: `timestamp + "." + тело запроса`
- Заголовок `X-Voice-Timestamp` содержит Unix timestamp момента отправки
- Бот верифицирует подпись и проверяет, что timestamp в пределах ±5 минут (защита от replay-атак)
- Секрет выдаётся при регистрации бота в Developer Portal

### Polling (v1, для разработки)

Альтернатива вебхуку для локальной разработки — бот сам опрашивает endpoint платформы: `GET /api/bots/me/interactions/poll`. Не требует публичного URL. **Только для тестирования**, не рекомендуется для production.

## Scopes — что может делать бот

| Scope | Что разрешает |
|-------|--------------|
| `send_messages` | Отправлять сообщения в каналы, где включён |
| `send_dm` | Писать пользователю в DM — только в ответ на его действие, не инициативно |
| `read_members` | Видеть список участников спейса |
| `manage_roles` | Выдавать/забирать роли ниже бота по иерархии |
| `manage_channels` | Создавать/архивировать каналы |
| `read_messages` ⚠️ | Читать все сообщения в каналах (привилегированный scope, требует обоснования) |

Scope `read_messages` — привилегированный: при добавлении бота пользователь видит явное предупреждение. Предназначен для модерационных ботов.

## Пост-v1 backlog

- **Socket Mode / Gateway** — бот подключается к платформе по WebSocket, не нужен публичный URL; production-альтернатива webhook для команд за NAT или в корпоративных сетях
- **Context menu commands** — команды по правому клику на сообщении или пользователе
- **Incoming Webhooks** — простой URL для отправки сообщений без OAuth; для CI/CD, мониторинга, внешних сервисов
- **Bot DM по инициативе** — scope для уведомлений (opt-in от пользователя); сейчас бот может писать только в ответ
- **Каталог ботов** — публичный App Directory со страницами ботов, отзывами, категориями
