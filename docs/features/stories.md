# Stories — сторис

## Общее

Сторис — короткий контент (фото / видео / текст), живущий 24 часа в ленте, после чего уходящий в личный архив. Видимость контролируется стандартными настройками приватности.

Базовая механика — как в Телеграме. Гейминг-специфика выделена отдельно.

---

## Форматы контента

| Формат | Описание |
|--------|----------|
| Фото | Статичное изображение |
| Видео | До 60 секунд |
| Текст | Текст на цветном фоне |
| Клип | Короткий игровой момент — отличается от обычного видео тем, что редактор предлагает инструменты трима и выделения фрагмента |

---

## Редактор

- Текстовые оверлеи (шрифты, цвет, размер, позиция)
- Стикеры
- Дудл (рисование поверх)
- Фильтры и эффекты (базовый набор — бесплатно, расширенный — по подписке)
- Для видео/клипа: трим прямо в редакторе

---

## Гейминг-специфика

### Game tag
На стори можно прикрепить игру из каталога — как геотег, но игра. Отображается как плашка поверх контента. По тапу — страница игры в каталоге (или лента стори с этой игрой).

### "Ищу пати"
Специальный тип стори-объявления. Отличия от обычной стори:
- Визуально выделена (специальный стиль карточки в ленте)
- Содержит кнопку "Присоединиться" / "Написать"
- По тапу открывается форма матчмейкинг-запроса (предзаполненная данными из стори)
- Интегрируется с системой матчмейкинга — см. [matchmaking.md](matchmaking.md)

### Авто-генерация после матча
После завершения матча система предлагает создать стори:
- Предзаполненный шаблон: игра, результат (победа / поражение), упоминания партнёров
- Пользователь может отредактировать или отклонить предложение
- Партнёры по матчу получают нотификацию при публикации стори с их упоминанием

## Просмотр

### Где отображаются сторис

| Место | Поведение |
|-------|-----------|
| Список контактов / чатов | Кольцо вокруг аватара при наличии активной стори (как в Телеграме) |
| Профиль пользователя | Тап по аватару — открывает просмотрщик стори |

### Навигация в просмотрщике
- Тап вправо / влево — следующая / предыдущая стори
- Свайп вниз — закрыть

---

## Интерактивность

### Реакции
- Эмодзи-реакции (как в Телеграме)
- Видны только автору стори

### Ответы (реплаи)
- Только приватные — уходят в ЛС между автором и ответившим
- Публичных комментариев нет

### Упоминания (@username)
- Можно упомянуть других пользователей
- Упомянутый получает нотификацию
- В своей стори получивший может репостнуть исходную стори с подписью

---

## Жизненный цикл стори

```
Публикация → 24 часа в ленте → уход в архив автора
```

- Активная стори (< 24ч) — видна в контакт-листе и профиле по настройкам приватности
- По истечении 24ч — стори исчезает из ленты, но сохраняется в личном архиве автора
- Архив хранится **30 дней**, затем удаляется (и медиа из хранилища тоже)
- Архив — только для автора, если не вынесена в Highlights
- Лимиты медиафайлов стори — те же что в [file-storage.md](file-storage.md)

---

## Архив и Highlights

### Архив
- Все опубликованные стори сохраняются в архив
- Архив доступен только самому пользователю
- Из архива можно вынести стори в Highlights

### Highlights
- Именованная коллекция стори из архива, закреплённая на профиле
- Видна всегда (не истекает)
- Своя настройка приватности — независимая от исходной стори
- Отображается на профиле как иконки под биографией (как в Instagram)

---

## Приватность

Поле "Стори" уже описано в [privacy.md](privacy.md). Используется стандартный контрол выбора аудитории.

Дополнительно:
- Highlights имеют **отдельную** настройку приватности (не наследуют от архивной стори)
- "Ищу пати"-стори всегда имеет видимость ≥ настроек стори (нельзя сделать её более закрытой, чем обычная стори, так как она функциональная)

---

## Просмотры

- Счётчик просмотров отображается под стори (виден только автору)
- Список "кто посмотрел" — доступен только автору, пока стори активна (24ч)
- После ухода в архив список просмотров не сохраняется (только счётчик)
