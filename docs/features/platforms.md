# Platforms — платформы и клиенты

## Приоритет

1. Web — быстрый выход без апп-сторов
2. Windows — основная аудитория
3. Android / iOS — после стабилизации

macOS и Linux — поддерживаются Flutter, но не в приоритете; реализуются по запросу.

Flutter — единая кодовая база для всех платформ.

## Ограничения веб-версии

- Нет шары системного аудио (техническое ограничение браузера)
- Нет глобального PTT-хоткея (вне фокуса вкладки)

## Мобайл

- Полный функционал, как на десктопе
- Шара экрана — не в MVP, реализуется позже
- Войс продолжается в фоне при сворачивании (см. [voice-chat.md](voice-chat.md))

## Одновременное использование нескольких устройств

- Пользователь может быть залогинен на нескольких устройствах одновременно; один активный войс-чат на профиль (одно устройство = один войс, переключение — через UI)
- Сообщения синхронизируются на всех устройствах в реальном времени

## Оверлей в игре

- Планируется (как Discord overlay — кто говорит, управление войсом без выхода из игры)
- Реализуется после MVP

## Service Ownership

- Feature owner: клиентские приложения (Flutter Web/Desktop/Mobile)
- Backend ограничения платформ: `Voice Service` (voice concurrency), `Notification Service` (platform push specifics)

## Enforcement path

1. Клиент определяет platform capabilities и скрывает/блокирует недоступные функции.
2. `Voice Service` валидирует ограничение одного активного voice-session на профиль.
3. `Notification Service` применяет платформенные каналы доставки (FCM/APNs/WebPush).
