# Privacy — настройки приватности

## Общее

Настройки приватности применяются ко всем сущностям приложения. Каждое чувствительное поле имеет собственный контрол видимости/доступа.

При создании профиля пользователь выбирает один из трёх пресетов с предустановленными настройками приватности. Пресет можно поменять позже вручную.

### Таблица дефолтов по пресетам

ДД = Друзья друзей, УС = Участники спейса, Г = Гостевые аккаунты

| Поле                     | Личный      | Игровой     | Рабочий     |
|--------------------------|-------------|-------------|-------------|
| Онлайн-статус            | Друзья      | Все         | УС          |
| Статус «в игре»          | Друзья      | Все         | Никто       |
| Рейтинг ММ               | Друзья + ДД | Все         | Никто       |
| Телефон                  | Никто       | Никто       | Никто       |
| Стори                    | Друзья + ДД | Все         | Никто       |
| Поиск по телефону        | Друзья      | Друзья      | Никто       |
| Начинать диалог          | Друзья + ДД | Все         | УС + Друзья |
| Звонить                  | Друзья      | Друзья + ДД | УС + Друзья |
| Приглашать в чаты/спейсы | Друзья      | Друзья + ДД | Никто       |
| Присылать файлы          | Друзья + ДД | Друзья + ДД | УС + Друзья |
| Присылать голосовые      | Друзья      | Друзья + ДД | УС + Друзья |
| Приглашать в друзья      | Все         | Все         | УС          |

Примечание: «Все» включает гостей; конкретные группы (Друзья, ДД, УС) — гостей не включают. В игровом пресете «Все» означает, что гостевые аккаунты видят онлайн-статус и стори. Пользователь может явно добавить/убрать «Гостевые аккаунты» для любого поля.

**Логика пресетов:**
- **Личный** — полуоткрытый, для реальных друзей
- **Игровой** — открыт к незнакомцам по игровым осям; оптимален для матчмейкинга
- **Рабочий** — закрыт снаружи, открыт внутри спейса; телефон и игровые данные скрыты

## Контрол выбора аудитории

Мультиселект со следующим набором опций (единый для всего приложения):

| Опция             | Поведение                                                                             |
|-------------------|---------------------------------------------------------------------------------------|
| **Все**           | Шорткат вверху списка — снимает «Никто», выбирает все остальные опции, включая гостей |
| **Никто**         | Шорткат вверху списка — снимает весь выбор                                            |
| Друзья            | Стандартная опция                                                                     |
| Друзья друзей     | Стандартная опция                                                                     |
| Участники спейса  | При выборе появляется второй мультиселект — список спейсов профиля                    |
| Гостевые аккаунты | Временные аккаунты без регистрации; по умолчанию не включены в конкретные группы      |

«Все» и «Никто» стоят первыми в списке и являются взаимоисключающими шорткатами, а не рядовыми пунктами.

Итоговая аудитория — **объединение** всех выбранных групп.

**Поведение гостей:** при выборе «Все» гостевые аккаунты включены. При выборе конкретных групп (Друзья, Друзья друзей, Участники спейса) — гости не входят в аудиторию, если «Гостевые аккаунты» не выбраны явно.

### Выбор спейсов

Если выбрана опция «Участники спейса», появляется мультиселект со спейсами, в которых состоит пользователь. Доступ получают участники выбранных спейсов.

## Принятые решения

- **Друзья друзей**: 1 уровень глубины — только прямые друзья друзей; рекурсия — проблема производительности и смысла
- **UX блокировки**: заблокированный не находит пользователя в поиске; профиль показывает "пользователь недоступен"; факт блокировки не раскрывается (как Telegram)
- **Блокировка — односторонняя видимость**: A заблокировал B → A не видит сообщения B в общем спейсе, B не отображается в списке участников для A; B при этом видит сообщения A (блокировка не взаимная по умолчанию). В личной переписке — DM недоступен с обеих сторон.

## Поля с настройками приватности

### Видимость данных

Кто видит информацию:

- Онлайн-статус
- Статус «в игре»
- Рейтинг ММ
- Телефон
- Стори

### Разрешения на действия

Кто может совершать действия:

- Поиск профиля по номеру телефона
- Начинать диалог
- Приглашать в групповые чаты и спейсы
- Звонить
- Присылать файлы
- Присылать голосовые сообщения
- Приглашать в друзья

## Service Ownership

- Source of truth: `User Service` (privacy settings per profile)
- Social graph inputs: `Social Service` (friends, friends-of-friends, blocks)
- Space audience inputs: `Space Service` (участники выбранных спейсов)

## Enforcement path

1. Клиент сохраняет privacy rules через `API Gateway` в `User Service`.
2. `User Service` вычисляет эффективные аудитории с учётом профиля.
3. Action-oriented сервисы (`Messaging`, `Voice`, `Story`, `Matchmaking`) запрашивают/получают policy-решение перед выполнением действия.
4. `Social Service` и `Space Service` поставляют membership/relationship факты для финального allow/deny.
