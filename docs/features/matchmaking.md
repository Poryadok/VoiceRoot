# Matchmaking — матчмейкинг

## Типы матчмейкинга

| Тип           | Статус        | Особенность                                                                 |
|---------------|---------------|-----------------------------------------------------------------------------|
| Глобальный    | Первая версия | Параметры вводятся вручную, без верификации данных                          |
| Внутри спейса | Позже         | Спейс задаёт параметры и верификацию; меньше аудитория, но лучше фильтрация |

Глобальный — приоритет. Пересмотреть баланс когда аудитория вырастет до уровня Discord.

**Space-level ММ** работает только на мастер-сервере. Федерат-ноды не имеют собственного ММ — пользователи федерат-нод участвуют в ММ через синхронизацию с мастером. Локальный ММ на федерат-нодах — возможная фича в будущем.

## UI — расположение

- **Десктоп**: панель справа
- **Мобайл**: отдельная вкладка

## Пати

- **Соло**: один пользователь
- **Группа**: все участники войс-чата в момент запуска ММ автоматически считаются пати
- Запустить поиск может любой участник войса
- Если кто-то вошёл или вышел из войса во время поиска → ММ сбрасывается, предлагается начать поиск заново

## Параметры поиска

Параметры задаются отдельно для каждой игры. Продуктовая спека каталога (страница игры, UX) — [game-catalog.md](game-catalog.md). Хранение в ММ, модерация и пользовательское добавление — в разделе [Каталог игр](#каталог-игр) ниже.

**Пример (Valorant):**
- Игра: Valorant
- Режим: (определяет количество слотов — например, 5v5 → 5 мест)
- Свой ранг
- Ранги, которые ищешь

Для глобального ММ — поля заполняются вручную, верификация рангов отсутствует (на доверии).

### Алгоритм подбора

Матч засчитывается, когда нашлась группа с **точным попаданием в критерии обеих сторон**. Критерии бывают:
- **Точное значение** — e.g., конкретный регион
- **Диапазон** — e.g., ранг 1500–1600; совпадение требуется в пересечении диапазонов

## Каталог игр

Поля карточки игры и сценарии в клиенте — в [game-catalog.md](game-catalog.md). Здесь — модель данных и правила в контексте очередей ММ.

Игры хранятся как JSON-документы, описывающие ММ-критерии. Несколько примеров будут добавлены при запуске (Dota 2, CS2, Valorant, PUBG).

### Модерация каталога

- Добавление новой игры создаёт запрос на модерацию — не попадает в каталог сразу
- На старте модерирует владелец вручную; позже — назначенные модераторы
- Защита от дублей и спама

### Добавление новой игры пользователем

Если нужной игры нет в списке — кнопка "Добавить игру" открывает конструктор:

- **Режимы**: добавить режим игры (каждый режим задаёт количество слотов; например, "5v5" → 5 мест в пати)
- **Числовой критерий**: название + целое число (например, "ELO рейтинг")
- **Список / enum**: название + перечень значений (например, "Ранг": Bronze, Silver, Gold...)
- Для каждого критерия указывается: применяется ли к "себе" и/или к "кого ищу"

**Правила:**
- Редактировать созданную игру нельзя — как настроен первым, так и остаётся
- Если несколько пользователей добавят одну игру одновременно — конфликт разрешается первым поступившим

## Ограничения и настройки поиска

- **Одновременных поисков**: только по одной игре одновременно на пользователя
- **Регион/пинг**: регион как **обязательный** параметр ММ, не ослабляется при долгом поиске; на старте — выбор вручную (Россия / Европа / Азия); автоопределение по IP — позже
- **Кросс-региональный поиск**: не поддерживается; матч возможен только в рамках выбранного региона

## Флоу поиска

1. Пользователь (один или из войса) запускает поиск с заданными параметрами
2. Во время поиска можно пользоваться приложением в обычном режиме
3. Когда матч найден → **попап у всех участников**: "Матч найден — Принять / Отклонить"
4. Логика при отказе:
   - Кто-то из **своей пати** отклоняет → поиск **сбрасывается** для всей пати
   - Все свои приняли, но кто-то из **чужой пати** отклоняет → своя пати **продолжает поиск**
5. **Промежуточное уведомление:** через 15 минут без результата → уведомление "Долго ищем. Попробуйте изменить параметры поиска"; поиск продолжается
6. **Таймаут:** через 30 минут без результата → уведомление "Найти не удалось, попробуйте снова"; поиск останавливается
7. Все приняли → создаётся временная комната

## Временная комната

- Войс + прикреплённый текстовый чат
- Автоудаляется после того как все вышли
- Запись остаётся в истории ММ

## История матчмейкинга

- Список комнат, в которых участвовал пользователь
- Для каждой комнаты — список участников
- Из истории можно:
  - Добавить пользователя в друзья
  - Забанить пользователя (только для ММ)

## Оценка тиммейтов

- Появляется при **выходе из комнаты**
- Шкала: 1–5 звёзд
- Можно пропустить (для всех или для отдельных участников)
- При оценке 1 или 2 — предложение забанить пользователя
- Средний рейтинг пользователя **виден в профиле**

## Бан

- Забанить можно при оценке (1–2 звезды) или через правую кнопку мыши на пользователе
- Бан только в рамках ММ: забаненный пользователь больше не будет матчиться к тебе
- Не влияет на мессенджерную часть (переписка, войс и т.д.)

## Social Discovery — "Ищу пати" в Сторис

Пользователь может опубликовать стори с объявлением о поиске пати. Это **не активная запись в ММ-очередь** — просто объявление.

### Структура стори

- Параметры игрока: игра, ранг, роль и т.д.
- ММ-параметры: что ищет (ранг, режим и т.д.)
- Две кнопки: **Присоединиться** и **Пригласить**

### Кнопка "Присоединиться"

Зритель хочет войти в пати автора:

1. Если в профиле зрителя заполнены данные по этой игре — автору стори приходит уведомление:
   > "@username хочет присоединиться. Ранг: [X]"
2. Автор принимает → образуется пати, ММ перезапускается
3. Несколько человек нажали → автору приходит несколько сообщений, он выбирает

### Кнопка "Пригласить"

Зритель приглашает автора стори в свою пати:

1. Автору стори приходит уведомление:
   > "@username приглашает вас в пати. Участников: N. Параметры: [...]"
2. Автор решает — принять или нет

### Неактивное состояние

Если автор уже нашёл пати / вышел из ММ / набрал полную пати — обе кнопки становятся **неактивными**.
Если кто-то успел нажать до деактивации → ошибка: "Объявление больше не актуально".
