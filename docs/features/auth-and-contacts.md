# Auth & Contacts — регистрация и контакты

## Способы регистрации

| Способ         | Статус       | Примечание                                                                                        |
|----------------|--------------|---------------------------------------------------------------------------------------------------|
| Номер телефона | По умолчанию | Основной способ                                                                                   |
| Email          | Опционально  | Бейдж "без телефона" на аккаунте; возможные ограничения в будущем (например, на инициирование DM) |

## Гостевая учётка

- При открытии веб-версии без входа автоматически создаётся временный аккаунт
- Гость может делать всё, что разрешено гостям в конкретном чате (настраивается на уровне чата/спейса)
- Гость **не может**:
  - Инициировать разговор (DM, войс-звонок)
  - Самостоятельно вступать в чаты и спейсы — только по инвайту
  - Добавлять в друзья
- При регистрации история гостя **переносится** на новый аккаунт
- У каждого спейса и чата есть настройка: разрешать ли вход гостям

### Видимость для владельца профиля (настройки приватности)

В настройках приватности профиля — "Гостевые аккаунты" является отдельной опцией аудитории. Позволяет отдельно регулировать, что видят гости, независимо от настроек для зарегистрированных пользователей.

Применяется к:
- Онлайн-статус
- Статус «в игре»
- Рейтинг ММ
- Стори

Действия (DM, звонок, приглашение в друзья) гостям недоступны вне зависимости от настроек — настройка этих полей не применяется к гостям.

## Идентификатор пользователя

| Тип                       | Формат                                                  | Условие                                                                                       |
|---------------------------|---------------------------------------------------------|-----------------------------------------------------------------------------------------------|
| Стандартный               | `@username#1234`                                        | Бесплатно                                                                                     |
| Премиум                   | `@username`                                             | Подписка                                                                                      |
| Верифицированная личность | `✅username` (в UI — системная иконка, не символ в нике) | Доказательство через OAuth: Twitch Partner / YouTube YPP — [verification.md](verification.md) |
| Компания / юрлицо         | `®companyname` (в UI — системная иконка)                | Владение доменом: TXT в DNS + ручной fallback — [verification.md](verification.md)            |

Дискриминатор `#1234` у стандартных аккаунтов — случайный (возможность выбора TBD).

## Контакты по телефону

- Механика как в Telegram / WhatsApp: пользователь разрешает доступ к телефонной книге, приложение показывает кто из контактов уже в Voice
- Работает в обе стороны: если пользователь ввёл номер и разрешил поиск — его можно найти

## Сессии и безопасность

- **Токены**: JWT access token (15 мин) + opaque refresh token (30 дней) в DB
- **Таблица**: `refresh_tokens(id, account_id, token_hash, device_info, expires_at, created_at, revoked_at)`  
  (`user_id` в JWT claim историческое имя, значение = `account_id`)
- **Инвалидация**: смена пароля → удалить все refresh tokens; logout → удалить один
- **Досрочный отзыв**: Redis blacklist для access token до истечения TTL
- **Сессии**: неограниченно одновременно → страница "Активные устройства" в настройках
- **Восстановление пароля**: через email (ссылка/код на почту)
- **Rate limiting авторизации**: 5 попыток / 15 мин с одного IP → блок 15 мин (экспоненциально); OTP: 3 попытки / 10 мин, новый код не чаще раза в минуту; реализация — Redis sliding window

## Гостевой аккаунт — жизненный цикл

- Истекает через 30 дней после последнего онлайна
- Ограничений по количеству чатов/сообщений нет

## Удаление аккаунта / GDPR

- Soft delete: поле `deleted_at` на таблице `accounts` в `auth_db`
- Данные остаются в основной БД (нужны для антискама и 152-ФЗ)
- Отдельный архив не нужен на старте

### Что видят другие после удаления аккаунта

- **История чата**: на устройствах, где чат уже загружен, сообщения удалённого пользователя остаются. В конце переписки — системное сообщение "Пользователь удалён". Отправлять новые сообщения в этот чат нельзя.
- **Новые устройства**: чат с удалённым пользователем не подгружается (не отображается в списке).
- **Профиль по ссылке**: страница "Аккаунт удалён" (без данных).
- **В списках участников спейса**: пользователь не отображается после удаления.
