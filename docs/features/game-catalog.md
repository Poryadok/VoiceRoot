# Game Catalog — каталог игр

Очереди, пати и логика матча — в [matchmaking.md](matchmaking.md).

## Концепция

Каталог игр — справочник, из которого пользователи выбирают игру при запуске матчмейкинга. Каждая игра в каталоге имеет страницу с описанием и конфигурацией под матчмейкинг.

---

## Страница игры

| Поле                | Описание                                                                        |
|---------------------|---------------------------------------------------------------------------------|
| Название            | Официальное название игры                                                       |
| Логотип / обложка   | Изображение                                                                     |
| Жанр                | Экшн, MOBA, Шутер, Стратегия и т.д.                                             |
| Платформы           | PC, Console, Mobile                                                             |
| Регионы             | Для каких регионов доступна                                                     |
| Режимы матчмейкинга | Набор режимов (5v5, Solos, Duos и т.д.)                                         |
| Роли                | Список ролей для данной игры (например, для Dota 2: Carry, Mid, Support и т.д.) |
| Ранговая система    | Список рангов (self-reported пользователем)                                     |

## Режимы и конфигурация матчмейкинга

Для каждой игры задаётся набор режимов. Каждый режим описывает структуру матча:

```
Режим "5v5 Ranked":
  - Игроков в матче: 10 (5 на каждую сторону)
  - Размер пати: 1–5
  - Роли: обязательны / опциональны
  - Ранг: обязателен / опционален
```

Конфигурация режимов задаётся при добавлении игры в каталог.

---

## Добавление игры

### Кто может добавлять

- Глобальные модераторы платформы — через Admin Panel

### Процесс

1. Модератор открывает форму добавления игры
2. Заполняет поля: название, логотип, жанр, платформы, регионы
3. Настраивает режимы матчмейкинга и роли
4. Публикует — игра становится доступна в каталоге

### Заявки от пользователей (пост-v1)

- Пользователь может подать заявку на добавление игры через форму
- Модератор рассматривает заявку и публикует или отклоняет

---

## Дедупликация

- Перед добавлением модератор проверяет отсутствие дубля по названию
- Поиск по каталогу при добавлении подсвечивает похожие игры
- Привязка к внешним ID (Steam AppID и т.д.) — пост-v1, используется для автоматической дедупликации

---

## UX в клиенте

- Поиск по каталогу при запуске матчмейкинга — строка поиска с автодополнением
- Популярные игры — показываются первыми (по количеству активных очередей)
- Недавние игры пользователя — отдельный раздел "Недавнее" для быстрого доступа
- Страница игры открывается при тапе на карточку в каталоге или из профиля (история матчей)

---

## Пост-v1 backlog

- Заявки на добавление игр от пользователей
- Верификация ранга через Steam API / Riot API и т.д.
- Страница игры с лидербордом пользователей платформы
- Подписка на игру — уведомление при появлении новых режимов или событий
