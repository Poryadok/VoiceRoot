# Accessibility (a11y) — доступность

Базовый уровень доступности взят как пересечение практик Discord и Telegram. Полный WCAG 2.1 AA не является целью v1, но закладываем основу.

## Клавиатурная навигация

### Глобальные шорткаты (Desktop)
| Действие                                        | Шорткат                   |
|-------------------------------------------------|---------------------------|
| Перейти к следующему непрочитанному каналу/чату | `Alt+↓` / `Alt+↑`         |
| Перейти к строке поиска                         | `Ctrl+K`                  |
| Перейти к строке ввода сообщения                | `Escape` (из любого поля) |
| Открыть настройки                               | `Ctrl+,`                  |
| Переключить PTT (Push-to-Talk)                  | Настраиваемая клавиша     |

### Навигация по интерфейсу
- Все интерактивные элементы достижимы через `Tab` / `Shift+Tab`
- Порядок фокуса следует визуальному порядку (left-to-right, top-to-bottom)
- Видимый индикатор фокуса (outline) не скрывается — `:focus-visible` на веб
- Модальные окна и попапы ловят фокус (focus trap) и возвращают его при закрытии

### Навигация по списку сообщений
- `↑` / `↓` для перемещения между сообщениями (как в Discord)
- `Enter` на выбранном сообщении — открыть контекстное меню
- `R` — ответить на выбранное сообщение
- `E` — добавить реакцию

## Screen Reader

### Web (Flutter Web / потенциальный React-клиент)
- Семантическая HTML-разметка: `<nav>`, `<main>`, `<section>`, `<button>`, `<input>`
- ARIA-роли и лейблы для всех иконок и кнопок без видимого текста
- `aria-live` регион для новых сообщений (режим `polite` — не перебивает чтение)
- Атрибут `aria-label` на аватарах ("Аватар [имя пользователя]")
- Статусы присутствия аннотированы текстом ("онлайн", "не активен")

### Flutter (Mobile / Desktop)
- Виджет `Semantics` на все кастомные компоненты
- `excludeSemantics` для декоративных элементов
- `MergeSemantics` для составных элементов (аватар + имя + статус → одна semantic node)
- Тестирование с TalkBack (Android) и VoiceOver (iOS)

## Визуальная доступность

### Контрастность
- Основной текст на фоне: минимум **4.5:1** (WCAG AA)
- Крупный текст (≥18px / ≥14px bold): минимум **3:1**
- Интерактивные элементы (кнопки, иконки): минимум **3:1** от фона

### Темы
- Светлая и тёмная тема (системная + ручная — см. navigation.md)
- Высококонтрастный вариант (High Contrast) — дополнительная тема, переключается в настройках
  - Аналог: Discord "High Contrast" и системная тема Windows/macOS

### Масштабирование текста
- Приложение адаптируется к системному размеру шрифта (100% — 200%)
- Лейауты не ломаются при размере текста ×1.5 — проверяется как часть UI-тестов
- Нет fixed pixel heights для контейнеров с текстом

### Цвет как единственный индикатор
- Информация не передаётся только цветом — всегда сопровождается иконкой или текстом
- Пример: статус "онлайн" = зелёная точка + alt-текст "онлайн", не только цвет

## Уменьшение движения

- Уважаем `prefers-reduced-motion` на вебе → анимации заменяются на мгновенные переходы
- В настройках приложения: явный тумблер "Reduced motion" (Flutter не читает системный параметр автоматически на всех платформах)
- Под ограничение подпадают: анимация открытия/закрытия панелей, flying emoji в реакциях, анимированные стикеры/GIF — последние переходят в статичный первый кадр

## Поддержка платформенных механизмов

| Платформа        | Поддержка                            |
|------------------|--------------------------------------|
| iOS VoiceOver    | да — через Flutter Semantics         |
| Android TalkBack | да — через Flutter Semantics         |
| Windows Narrator | да — через Flutter Semantics + MSAA  |
| macOS VoiceOver  | да — через Flutter Semantics         |
| Web (NVDA, JAWS) | да — через ARIA + семантический HTML |

## Что не входит в v1

- WCAG 2.1 AAA (повышенные требования к контрасту и т.д.)
- Брайлевский дисплей (не тестируем, но не блокируем)
- Аудиодескрипция для видео-контента
- Субтитры для голосовых сообщений (авто-транскрипция) — отложено

## Тестирование

- Автоматические lint-проверки: `flutter analyze` с включённым `avoid_print` и accessibility-правилами
- Ручное тестирование с TalkBack и VoiceOver перед каждым релизом
- Axe DevTools (или аналог) для веб-версии как часть CI
