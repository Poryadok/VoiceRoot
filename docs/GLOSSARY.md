## Чаты и спейсы

### Чат

**Чат** — общее обозначение места для общения (абстракция). В коде и API всегда уточняется типом (личный, группа, канал и т.д.).

### Личный чат

**Личный чат** — общение **один на один** (два участника). Не входит в состав спейса как его «содержимое»; в UI обычно отдельный список (ЛС).

### Групповой чат

**Групповой чат** — место для общения **нескольких** участников (не 1:1). Делится на:

| Вид                         | Смысл                                                                                                                                                                                                                     |
|-----------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Текстовый групповой чат** | Поток **текстовых** сообщений (группа или канал — см. ниже)                                                                                                                                                               |
| **Голосовая комната**       | Совместный голос (и опционально видео): в **спейсе** — постоянные голосовые комнаты; у **текстовой группы** — при необходимости **временная** комната, привязанная к группе — см. [voice-chat.md](features/voice-chat.md) |

## Группа и канал (текст)

Оба — подтипы **текстового группового** чата. Различие — **модель основного потока сообщений**, а не «кто как состоит в чате»: ограничения доступа, видимость, право писать и прочие правила задаются **настройками конкретного чата** (и ролями там, где это уместно) и **не определяют** тип «группа / канал».

|                          | **Группа** (`group`)                                                                                                             | **Канал** (`channel`)                                                                                                              |
|--------------------------|----------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------|
| **Основной поток**       | Сообщения в ленту пишут **участники** (от своего имени).                                                                         | В основной ленте публикует **сущность канала** (официальные посты); участники обсуждают в **тредах** у постов, а не в общей ленте. |
| **Поведение в продукте** | Треды — по настройкам группы; см. [text-chat.md](features/text-chat.md).                                                         | У каждого поста — тред; `@here` / `@everyone` — см. [text-chat.md](features/text-chat.md) и [roles.md](features/roles.md).         |
| **Где живёт сущность**   | `chat_db.chats` с `type = group`; при спейсе — **`space_id`** → `spaces.id`; возможна группа **вне спейса** (`space_id` пустой). | `space_db.channels` с `type = text` (отдельной строки в `chats` для этого потока **нет**); канал **только внутри спейса**.         |
| **Messaging**            | `chat_type = group`, `chat_id = chats.id`.                                                                                       | `chat_type = channel`, `chat_id = channels.id`.                                                                                    |

## Голосовой групповой чат

**Голос в спейсе** — голосовые комнаты как элементы структуры спейса: `space_db.channels` с `type = voice`, сессии Voice Service / LiveKit. Все подключённые участники общаются **от своего имени**; отдельного типа «вещание от имени канала» нет — только настройки прав (кто может войти, микрофон, модерация и т.д.) и фичи вроде командирского режима — см. [voice-chat.md](features/voice-chat.md).

**Голос у текстовой группы** — по сценарию «созвониться пати»: **временная** голосовая комната, **привязанная** к `chat_id` группы (не отдельный продуктовый тип «канал»).

Текстовые сообщения в Messaging к голосовой комнате спейса по умолчанию **не** привязываются к тому же `chat_id`, что у голоса (если нет отдельной фичи side-chat).

---

## Спейс

**Спейс** — контейнер **только для групповых чатов**: текстовые **каналы**, текстовые **группы** (с `space_id`), **голосовые комнаты** (элементы структуры спейса) — в любой комбинации; публичный или приватный по каталогу — см. [spaces.md](features/spaces.md).

**Личные чаты не являются содержимым спейса.** Модели «спейс из одних только ЛС» нет — личные чаты живут снаружи этой иерархии.

## Продукт → техника

| Продуктовый термин                   | `messaging_db.messages.chat_type` (если есть текст) | Первичная сущность в БД                                               |
|--------------------------------------|-----------------------------------------------------|-----------------------------------------------------------------------|
| Личный чат                           | `dm`                                                | `chat_db.chats`                                                       |
| Текстовая группа (вне или в спейсе)  | `group`                                             | `chat_db.chats` + `chat_members`; при спейсе — `space_id`             |
| Текстовый канал спейса               | `channel`                                           | `space_db.channels` (`type = text`)                                   |
| Голосовая комната в спейсе           | — (не Messaging)                                    | `space_db.channels` (`type = voice`) + Voice                          |
| Временная голосовая комната у группы | — (не Messaging)                                    | Voice-сессия / LiveKit room, связь с `chat_db.chats` (`type = group`) |

Детали ссылок между сервисами без FK: [DATA_MODEL.md](DATA_MODEL.md) §2.
