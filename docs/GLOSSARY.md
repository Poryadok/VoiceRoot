## Чаты и спейсы

### Чат

**Чат** — общее обозначение места для общения (абстракция). В коде и API всегда уточняется типом (личный, группа, канал и т.д.).

### Личный чат

**Личный чат** — общение **один на один** (два участника). Не входит в состав спейса как его «содержимое»; в UI обычно отдельный список (ЛС).

- БД: `chat_db.chats` с `type = dm`
- Messaging: `chat_type = dm`, `chat_id = chats.id`

### Групповой чат

**Групповой чат** — место для общения **нескольких** участников (не 1:1). Делится на:

| Вид                         | Смысл                                                                                             |
|-----------------------------|---------------------------------------------------------------------------------------------------|
| **Текстовый групповой чат** | Поток **текстовых** сообщений (и вложений) в Messaging.                                           |
| **Голосовой групповой чат** | **Голосовая комната** в спейсе (Voice / LiveKit); не путать с полем `chat_type` в `messaging_db`. |

## Группа и канал (текст)

Оба — подтипы **текстового группового** чата; отличаются **моделью доступа и членства**.

|                                         | **Группа** (`group`)                                                                              | **Канал** (`channel`)                                                                                                 |
|-----------------------------------------|---------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------|
| **Участники**                           | Явный список в **`chat_members`** (приглашения, состав подмножества спейса или без спейса).       | Доступ по **членству в спейсе и ролям** (Role + Space): кто видит канал — по правам, не отдельным «списком группы».   |
| **Где живёт сущность** (текст в спейсе) | `chat_db.chats` с `type = group` и, если группа привязана к спейсу, **`space_id`** → `spaces.id`. | `space_db.channels` с `type = text` (отдельной строки в `chats` для этого потока **нет**).                            |
| **Messaging**                           | `chat_type = group`, `chat_id = chats.id`.                                                        | `chat_type = channel`, `chat_id = channels.id`.                                                                       |
| **Поведение в продукте**                | Треды — по настройкам группы; см. [text-chat.md](features/text-chat.md).                          | Треды у сообщений; `@here` / `@everyone` — см. [text-chat.md](features/text-chat.md) и [roles.md](features/roles.md). |

**Группа** может быть **вне спейса** (`space_id` пустой) или **внутри спейса** (тот же тип `group`, заполненный `space_id`). **Канал** в этом смысле существует **только внутри спейса**.

## Голосовой групповой чат

В продукте соответствует **голосовому каналу** спейса: `space_db.channels` с `type = voice`, сессии — Voice Service / LiveKit. Текстовые сообщения в Messaging к этой сущности тем же `chat_id`, что у текстового канала, по умолчанию **не** привязываются (если нет отдельной фичи side-chat).

---

## Спейс

**Спейс** — контейнер **только для групповых чатов**: текстовые **каналы**, текстовые **группы** (с `space_id`), голосовые **каналы** — в любой комбинации; публичный или приватный по каталогу — см. [spaces.md](features/spaces.md).

**Личные чаты не являются содержимым спейса.** Модели «спейс из одних только ЛС» нет — личные чаты живут снаружи этой иерархии.

## Продукт → техника

| Продуктовый термин                  | `messaging_db.messages.chat_type` (если есть текст) | Первичная сущность в БД                                   |
|-------------------------------------|-----------------------------------------------------|-----------------------------------------------------------|
| Личный чат                          | `dm`                                                | `chat_db.chats`                                           |
| Текстовая группа (вне или в спейсе) | `group`                                             | `chat_db.chats` + `chat_members`; при спейсе — `space_id` |
| Текстовый канал спейса              | `channel`                                           | `space_db.channels` (`type = text`)                       |
| Голосовой групповой чат (канал)     | — (не Messaging)                                    | `space_db.channels` (`type = voice`) + Voice              |

Детали ссылок между сервисами без FK: [DATA_MODEL.md](DATA_MODEL.md) §2.
